使用node的express模块架设web服务器，实现以下功能：

1、根据用户请求的页面不同，向用户发送对应的页面文件，
（有index.html\login.html\register.html三个页面，分别表示首页、登录页、注册页），
请求其他页面时，统一发送404页，并设置响应码为404；


2、用户可以通过首页的“登录”按钮，跳转进登录页，用户输入用户名和密码后，把用户名和密码以GET请求的形式（用ajax发送get请求）发送给服务器，
由服务器从user.json文件中遍历是否存在该用户，不存在返回错误的json数据，存在则匹配对应的密码字段是否正确，错误则返回错误的json数据，否则返回正确的json数据；


3、用户通过登陆页面的“注册账号”按钮，跳转到注册页，输入用户名、密码、确认密码选项后，在前端验证两次输入的密码是否一致。一致时，
由ajax发送post请求给服务器，服务器在收到请求后，把该字段写入user.json文件中（写入前需判断该用户名是否存在，如果存在，向前端返回json数据，提示用户名已存在）


思路提示：
1、从user.json中读取数据，可使用管道流读取，然后转为对象，遍历对象下的属性值获取想要的数据

2、要把新用户写入user.json中，可以先取出user.json的数据，转为对象，然后向该对象中添加新数据，
把新对象覆盖写入进user.json文件即可


提示：

user.json文件格式：
  
用户名不存在时，向前端报错的json数据：
{
    "code":0,
    "des":"用户名不存在"
}

密码错误时，向前端报错的json数据：
{
    ”code":0,
    "des":"密码错误"
}

登录成功时，向前端报错的json数据：
{
    "code":1,
    "des":“登陆成功”
}
